JavaScript｜var、let 和 const 该如何选择？

发表于 2021-04-21 更新于 2021-04-22 阅读次数： Valine：

前言

刚开始学习 JavaScript 的时候，我相信大部分人和我一样，最先学到的几个语句不外乎是控制台打印语句console.log(); 和声明变量赋值语句 var num = 10;等等，可见声明变量多么重要。声明变量使用起来非常简单，但想要不出错却不容易。

什么是 let 和 const

let、const和var一样，都是声明变量的操作符。

为什么要用 let 和 const

var操作符在声明变量的时候会出现一些副作用，为此 ECMAScript6 专门更新了两个新的操作符let和const来避免出现使用var带来的意外。

使用 var 声明变量可能会出现的问题

声明提升

声明的变量只能是全局作用域或者是函数作用域

声明的变量可能会污染全局变量

该如何使用 let 和 const

let操作符声明的变量，可以重新赋值，const操作符则不行。

使用const操作符声明变量时必须同时初始化变量，let则可以只声明，不赋值。

let 和 const 优先使用哪个

我个人推荐优先使用const操作符，当一个变量需要重新赋值的时候，再使用let操作符。理由如下：

  • 执行效率高
  • 避免声明过的变量被意外覆盖

const 定义引用类型时的问题

文心｜序言

发表于 2021-04-21 阅读次数： Valine：

前言

写博客这个想法，不知道在脑海里来来回回多少遍了，总是没办法开始，不知道是因为自己的写作能力不行，还是从内心思想上的抗拒，所以我打算拜读一下豆瓣评分 9.0 的神作《文心》，提高一下写作能力，看看有没有可能开展我的博客之旅。

正文

第一个序言没有什么让我有所感慨的内容，却有一句让我回忆起自己学语文的时候：

即使就国文的知识来说，我们做中学生的时候所受的，不是一些繁繁碎碎，象从字纸簏里倒出来的知识，就是整部的《诗经》《书经》《易经》《礼记》，从陈年老书箱里搬出来，教我们读了做圣贤的。

读到“字纸簏”的时候不太理解，查过之后才知道是字纸篓的意思。

我上学的时候，虽然没有看四书五经那么夸张，但是也没好好学过，即使好好学了，也都还给老师了。尤其是工作了之后，更是很少阅读和写作，以至于现在的我一直有一种提起笔来就恐惧的心里，根本不知道该如何下笔。随着长大了才知道那些书为什么是圣贤书，只是当时我们还太小，需要经历过坎坷之后，才能从书中领悟一些些东西。

在第二个序言里有这么一段话：

因为思想也就存在词汇，字句，篇章，声调里； 中学生读书而只取其思想，那便是将书里的话用他们自己原有的词汇等等重记下来，一定是相去甚远的变形。

我读到这一句话的时候，发现自己就是其中所说的中学生，可能还不如中学生，总想着用自己的话来理解别人的意思，却从来没记住过别人的话里真正的意思。

突然想到自己读过蔡康永老师的《蔡康永的说话之道 2》，但读过之后给我留下的却只有一个书名，不知道是我自己的记忆力不好，还是根本没有好好去阅读一本书，希望以后的每一次阅读都能给自己留下来一些有用的东西。

后记

连这个后记怎么开头，我都琢磨了 5 分钟，2021 年第一次写了那么“长”的文章，不是 ctrl+c 和 ctrl+v，希望自己能坚持下去，更希望自己能留下点东西。
